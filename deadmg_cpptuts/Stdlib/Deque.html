std::deque

A deque is a double-ended queue. It lives in <deque>. Whilst other libraries tend to use more stringent specifications, the C++ deque is more akin to a vector. However, it has a number of advantages- for example, deques can insert or erase from both the front and the back very quickly, whereas for the vector fast insertion and erasure is available only at the back.

std::deque<int> x;
x.push_back(5);
x.push_front(7); // Adds 7 to the front of the deque- not available on vector
int i = x[0]; // Offers random access through [] like vector
x.pop_back();
x.pop_front(); // Erases the front item

Deques also offer random-access like vector and size(). However, unlike a vector, when pushing or popping at the front or back of a deque, it is guaranteed that your pointers and references are still valid. In addition, because deque does not require a single block of contiguous memory, it can be implemented in a much more intelligent fashion that does not require moving all the existing elements when the current capacity is exceeded. This advantage can save significant time when modifying a large deque. We will look at exactly how it is implemented later.

In many of the earlier pieces of C++ reference material, it was advised that vector be the default container of choice. However, deque is often superior, and more common advice now is to use deque by default, although vector is still the simplest container around.

As a final note, in the implementation shipped with Visual Studio, as of the time of writing (23/10/2012), deque has a fairly crippling implementation error that makes it almost unusably slow beyond trivial examples.
