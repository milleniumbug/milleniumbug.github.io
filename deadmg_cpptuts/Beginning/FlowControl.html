Flow Control

This is the simple act of making your program make decisions. All of our previous programs did things, but they couldn't change their behaviour- they always did the same thing every time. You could have replaced them with a sticky note on your monitor.

Now we will write a program whose effects are decided when we execute it. To do this, we will introduce a few new language constructs.

You may already be familiar with Boolean logic. This is the mathematics of just two states- one, or zero. Fortunately, we will not have to implement any boolean operations ourselves- they come with the language. The boolean type, for storing boolean values, is named bool. bool values can be true or false.

To make these booleans change our program's behaviour, we will need a language construct or three, and one function.

#include <iostream>
#include <cstdlib>
int main() {
    bool b = std::rand() % 2;

The std::rand function returns a random value. It's not particularly random or well-distributed, but good enough. We here use the modulo operator to force that value into the range 0 or 1, which is then implicitly converted (0 to false, 1 to true). Now the value of b, it's a mystery.

    if (b)
        std::cout << "b was true!\n";

If b is true, then the next statement is executed. If we want to execute multiple statements, we need curly braces (like those around main's body).

    else {
        std::cout << "b was false!\n";
        std::cout << "This makes me sad.";
    }

If b was false, the else is instead executed. Just like for if, we need braces here if we want to use more than one statement.

}

This simple program randomly determines which statement to output. In addition, we need to learn about the relational operators. There are six- <, <=, >, >=, !=, ==. Not all types support all operators, but the in-built integral types which we have already met all do. These are less than, less than or equal to, greater than, greater than or equal to, not equal to, or equal to, respectively. They all return boolean values indicating if the relation was true or false. We will also meet the while language construct. This construct executes the statement every time the condition is true.

#include <iostream>
#include <cstdlib>
int main() {
    while((std::rand() % 10) < 5)
        std::cout << "It was less than five!\n";
}

Every time the random function returns a value less than five (mod ten), the statement will be executed.

Finally, we meet the ! operator. This operator simply inverts the boolean value which was it's argument- if it was true, it becomes false, and if it was false, it becomes true. This allows us to simply express conditions like if something is not true by writing if(!cond) instead of if (cond).

After studying a necessary consequence of flow control, called scope, we will have the tools we need to tackle user input.

Indentation

In these samples, I haven't described indentation. I indented the body of main, but didn't explain why. Now we've met control flow, it's more obvious. The indentation allows us to quickly and easily determine where we are in the source code. If there is no indentation, then we are top-level. If there is just some indentation, then we are in main (or another function or class). Higher than that and we are in a condition- and where the indentation changes clearly reveals where the conditions are. It's worth developing and consistently using an indentation scheme, as it makes your code much easier to peruse. But there's a slight problem with this approach. Observe

int main() {
    if (false)
        std::cout << "In condition";
        std::cout << "Looks like it's in condition but actually isn't";
}

In the above code, it looks like the two statements are both part of the condition, but actually, the rules above clearly imply that the second statement is, infact, unconditionally executed. You can get automatic indentation tools which will indent your code for you, and many IDEs will automatically insert indentation and other characters to help you out.
